数据结构的分类

按照逻辑结构划分

集合结构：集合里面的元素是除了同属一个集合外，没有其他关系。
线性机构：元素之间是一对一的关系。
树形结构：元素和元素之间存在一对多的关系。
图形结构：元素与元素之间的关系可以多对多。

按照存储机构划分

顺序存储结构：数据元素存储在连续的存储单元内。
链式存储结构：把数据结构存在任意的存储单元，这些存储单元可以连续也可以不连续。

算法的复杂度

时间复杂度

表示方法：时间复杂度通常通过大O阶表示.
推导大O阶：用常数1替代运行时间中的所有加法常数；运行次数只保留最高阶项；如果最高阶项存在且不是1，则去除与这个项相乘的常数，得到的结果就是大O阶。
常见的时间复杂度:
$$O(1) < O(\log_n) < O(n) < O(n\log_n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)$$

空间复杂度

表示方法：算法程序在执行时候，需要的辅助单元所占的存储空间。通常一个算法的空间复杂度只考虑在运行过程中为局部变量分配的存储空间的大小，它包括为参数表中形参变量分配的存储空间和为在函数体中定义的局部变量分配的存储空间两个部分。

数据结构和算法的关系

数据就够和算法的关系好比食材和烹饪方法的关系，好的食材要用相对应好的烹饪方法才能做出美食，否则也只能是无米之炊。 对于一个算法，其时间复杂度和空间复杂度往往是相互影响的。当追求一个较好的时间复杂度时，可能会使空间复杂度的性能变差，即可能导致占用较多的存储空间；反之，当追求一个较好的空间复杂度时，可能会使时间复杂度的性能变差，即可能导致占用较长的运行时间。
