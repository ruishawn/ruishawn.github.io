为何升级

Python很多项目一直同时支持Python2和Python3，核心团队认为这样不利于Python的发展，Python大部分project将于2020年1月1日停止对Pyhton2的支持。基于Python开发的项目应该更多的考虑使用Python3开发或者兼容Python3。

升级

安装gcc编译工具

1

yum install -y gcc

下载资源包安装，本文安装的是3.6.6版本，安装其他版本在官网找对应的资源链接。

1
2
3

curl -L -O curl -L -O https://www.python.org/ftp/python/3.6.6/Python-3.6.6.tar.xz
xz Python-3.6.6.tar.xz
tar -xvf Python-3.6.6.tar

安装

1
2
3

make
make test
make install

make install 过程通常出现缺少一些依赖包的错误。解决办法：安装缺少的依赖包，然后 make&make install。

检验

命令行输入python3

1
2
3
4

Python 3.6.6 (default, Jul 16 2018, 14:34:50) 
[GCC 4.8.5 20150623 (Red Hat 4.8.5-28)] on linux
Type "help", "copyright", "credits" or "license" for more information.
>>>

如果想默认使用Python3，/usr/bin/python的软链接指向Python3的安装位置即可（不同的操作系统路径可能不同）。

1
2

rm /usr/bin/python
ln -s /usr/local/bin/python3 /usr/bin/python
